# Report Template Reference

Structure and required sections for `schema-report.md`.

## Document Structure

```markdown

> Generated by Kafka Schema Registry Analyzer on {date}
> Project: {project_name}

---

## Executive Summary

## Next Steps
```

**CRITICAL:** Use EXACT table format. Category column is MANDATORY.

```markdown

| 1 | {app_name} | {lang} | producer | {topics} | {serializer} | {yes/no} | {A/B/C/D/E} |
```

## RISKS Section

### auto.register.schemas=true

```markdown
### auto.register.schemas=true

> **Impact:** Schema evolution is uncontrolled. Breaking changes can be
> registered without review, potentially breaking all downstream consumers.

| # | App | File | Line | Topics Affected |
|---|-----|------|------|----------------|

```

### Custom Serializers Without Schema Registry

```markdown

> Schema Registry entirely. No contract enforcement, no compatibility
```

## Producer Upgrade Recommendations

**CRITICAL:** Each section MUST state "Category X" in heading.

```markdown
## Producer Upgrade Recommendations

### {App Name} ({Language}) — **Category {B|C|E}**

**Current state:**
- **Category:** {B|C|E} — {brief reason}
- Serializer: `{current_serializer}`
- Data model: `{class/file}`
- Topics: {topics}

**Recommended changes:**

1. **Add dependency:**
   {language-specific}

2. **Update serializer config:**
   {config changes}

3. **Add Schema Registry config:**
   {SR URL and auth}
```

**Example - CORRECT:**
```markdown
### order-processor (Python) — **Category C**

**Current state:**
- **Category:** C — Auto-registration enabled
- Serializer: AvroSerializer (with auto.register.schemas=true)
```

**Example - WRONG:**
```markdown
### order-processor (Python)  ← Missing "Category C"

**Current state:**
- Serializer: AvroSerializer with auto-registration ← No explicit category
```

## Migration Rollout Ordering

```markdown
## Migration Rollout Ordering

### Scenario 1: JSON data, no SR (Category B) — Producers First
1. **Upgrade producers** — Confluent serializer + `HeaderSchemaIdSerializer`
2. **Upgrade consumers** — Confluent deserializer

### Scenario 2: Already on SR (Category A→Header) — Producers Only
1. **Verify consumer versions** — Java 8.1.1+, Python 2.13.0+, etc.
2. **Upgrade producers** — add `HeaderSchemaIdSerializer`

### Scenario 3: Custom serdes → Confluent (Category E) — Consumers First
1. **Upgrade consumers** — composite deserializer (Java) or coordinated cutover
2. **Upgrade producers** — replace custom serializer
```

## Report Validation Checklist

Before completing, verify:

**✓ MUST HAVE:**
1. **Applications Discovered table** - Exact markdown table (not narrative sections)
   - Columns: # | App | Language | Role | Topics | Serializer | SR? | Category
   - Category column shows letter (A/B/C/D/E) for EVERY row
2. **Explicit category labels** throughout:
   - In table
   - In upgrade section headers: "app-name — Category X"
   - In terraform comments: `# Category: X`
   - When describing: "This is **Category C** because..."
3. **Terraform comment blocks** for every schema resource

**✓ VERIFY:**
- Search report for "Category A/B/C/D/E" - at least one MUST appear
- Each schema resource has comment block above it
- Applications Discovered starts with table, not prose
